WhirlyGlobe Component

The WhirlyGlobe Component is a framework you can use to put an interactive 3D globe in your app. You can provide base imagery, including networked tile data sets, labels, vectors, or even overlay UIView's.

The Component is entirely Objective-C and we've precompiled it so you can drop it directly into your app and go. For instructions on how to use it, please read the announcement .

To download the current release version of the WhirlyGlobe component, click the link above.

Maply Component

The Maply Component is a flat map display framework based on the same technology as WhirlyGlobe. They share the vast majority of their code and features and are, in fact, distributed as the same toolkit with different entry points for each.

Maply is being introduced with version 2.1 and can be obtained as part of the WhirlyGlobe-Maply distribution

The latest version of WhirlyGlobe-Maply is 2.1 and though it's still under development, it's fairly stable. You can check out the develop branch on github, or you can get a binary distribution of beta2.

WhirlyGlobe-Maply API

Where the Component is meant for simplicity, the API exposes it all in glorious detail. WhirlyGlobe-Maply is built with a combination of C++ and Objective-C. It pulls in a variety of open source projects, including Boost, Clipper and Eigen. It makes heavy use of the Standard Template Library. It's tricky if you're not familiar with Objective-C++.

If you can handle it, it's very powerful. WhirlyGlobe-Maply is built on a simple rendering engine that lives on the main thread. You add your content through a variety of data control layers which live in a separate thread. It's smart about data management and getting smarter all the time.

For a full list of features, check out the WhirlyGlobe 2.0 announcement and look at the WhirlyGlobe App in the app store.

To get started with the WhirlyGlobe-Maply API... well first reconsider using the Component. It's much easier. But if you're determined, clone the WhirlyGlobe repository and use the master branch. Check the README for dependencies, of which there are several, and get started.
