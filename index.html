WhirlyGlobe Component

The WhirlyGlobe Component is a framework you can use to put an interactive 3D globe in your app. You can provide base imagery, including networked tile data sets, labels, vectors, or even overlay UIView's.

The Component is entirely Objective-C and we've precompiled it so you can drop it directly into your app and go. For instructions on how to use it, please read the announcement .

To download the current release version of the WhirlyGlobe component, click the link above.

Maply Component

The Maply Component is a flat map display framework based on the same technology as WhirlyGlobe. They share the vast majority of their code and features and are, in fact, distributed as the same toolkit with different entry points.

Maply is in the current version (2.1) and you can get it as either source or a binary distribution as below.

WhirlyGlobe-Maply API

Where the Component is meant for simplicity, the API exposes it all in glorious detail. WhirlyGlobe-Maply is built with a combination of C++ and Objective-C. It pulls in a variety of open source projects, including Boost, Clipper and Eigen. It makes heavy use of the Standard Template Library. It's tricky if you're not familiar with Objective-C++.

It's all very powerful. WhirlyGlobe-Maply is built on a simple rendering engine that lives on the main thread. You add your content through a variety of data control layers which live in a separate thread. It's smart about data management and getting smarter all the time.

For a full list of features, check out the WhirlyGlobe 2.0 announcement and look at the WhirlyGlobe App in the app store.

The only reason to use the low level WhirlyGlobe-Maply API at this point is if you're doing something really specific and complex. If you are, I strongly suggest subclassing the appropriate view controller (e.g. the Component) and going from there. A lot of the tricky threading and resource issues are taken care for you. Most deployed apps are doing this.
