MaplyTileSource Protocol Reference

Declared in MaplyTileSource.h

Overview

Fill out this protocol and you can pass in your own data tile by tile. This protocol is used by the MaplyQuadImageTilesLayer to pull in image data per tile. This can be one or more images, they can be local, remote or even generated on the fly. It’s up to the object itself to return suitable data as requested or indicate failure (by returning nil).

The tile source should know its coordinate system, which is handed to Maply separately.

Instance Methods

imageForTile:

Fetch the image for a given tile.

- (id)imageForTile:(MaplyTileID)tileID

Return Value

Return a UIImage or an NSData containing raw PNG or JPEG data.

Discussion

You must fill in either imageForTile: or imagesForTile:numImages: if you’re doing more than one image per tile.

For this method, you can return either a full UIImage or an NSData containing an image that UIImage will recognize. Typically that’s a PNG or JPEG image of the typical size (128 or 256 pixels on a side).

If you fail to load the image, just return nil. At that point the paging won’t page in tiles below this image, assuming that image pyramid is truncated at that point.

imagesForTile:numImages:

Fetch an array of images for the given tile.

- (NSArray *)imagesForTile:(MaplyTileID)tileID numImages:(unsigned int)numImages

Return Value

An NSArray of UIImage or NSData objects.

Discussion

You must fill in either imageForTile: or imagesForTile:numImages: if you’re doing more than one image per tile.

This method is required if you’ve told the MaplyQuadImageTilesLayer that the depth is greater than 1. That is, it’s expecting more than one image per tile. You’d do this when you want to animate between them, for instance.

You must return an NSArray, but the array can contain UIImage or NSData entries (or both). For NSData we’re expecting a raw PNG or JPEG data, or anything that UIImage would interpret correctly.

tileIsLocal:

Can the given tile be fetched locally or do we need a network call?

- (bool)tileIsLocal:(MaplyTileID)tileID

Return Value

Return true for local tile sources or if you have the tile cached.

Discussion

We may ask the tile source if the tile is local or needs to be fetched over the network. This is a hint for the loader. Don’t return true in error, though, that’ll hold up the paging.
